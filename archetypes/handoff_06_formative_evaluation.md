# Archetype 6: Formative Evaluation Protocol Quality

**Purpose**: Evaluate formative evaluation plans, expert review protocols, usability test designs, and pilot study designs for methodological rigor and appropriateness.

**Evaluation Context**: Use this archetype when assessing:
- Instructional design evaluation plans (pre-implementation)
- Expert review protocols for e-learning modules or courses
- Usability test designs for learning interfaces
- Pilot study designs before full rollout
- Evaluation instruments (observation forms, interview guides, survey questionnaires)
- Think-aloud protocol designs for learning analytics
- Heuristic evaluation checklists for interactive learning environments
- Small group evaluation designs

---

## Rubric: 6 Dimensions (1-5 Scale)

### 1. Phase Coverage (1-5)

Does the evaluation plan include appropriate phases from Bordonaro's model? Not all phases required, but omissions should be justified.

| Score | Expert Review | One-to-One | Small Group | Field Trial | Rationale |
|-------|--------------|-----------|-----------|------------|-----------|
| **1** | Missing | Missing | Missing | Only phase | No justification for skipping |
| **2** | Mentioned | Minimal plan | Not planned | Only phases used | Weak justification |
| **3** | Planned | Planned | Mentioned | Optional | Moderate justification; logical gaps |
| **4** | Well-planned | Clear design | Clear design | Optional with rationale | Good justification; minor gaps |
| **5** | Comprehensive (3 expert types) | Detailed (mixed abilities) | Well-designed (8-12 learners) | Planned with N | Explicit rationale for all decisions |

**Bordonaro's Four Phases:**
1. **Expert Review**: SME, instructional design, media/technology experts evaluate (1-3 days typically)
2. **One-to-One**: 3-5 individual learners (above/average/below target level) work through with observer present; interview/think-aloud
3. **Small Group**: 8-12 learners work through in facilitated setting; group discussion, observation
4. **Field Trial**: Full implementation in target setting; larger sample; realistic conditions

---

### 2. Evaluator Selection (1-5)

Are the right evaluators chosen for each phase?

| Score | Expert Review | One-to-One Learners | Small Group Sample | Field Trial Sample | Documentation |
|-------|--------------|-------------------|------------------|------------------|----------------|
| **1** | Single expert only | Only average learners; no diversity | Unrepresentative sample | Too small; homogeneous | None |
| **2** | Two expert types; missing one | Two ability levels | Limited representation | Minimal sample | Minimal documentation |
| **3** | All three types mentioned | Mixed abilities present | Reasonably representative | Adequate N | Documented criteria |
| **4** | Three types clearly assigned | Three ability levels specified | Diverse, representative | Good N; stratified | Clear selection criteria |
| **5** | Three types, clear roles | Specifically recruited (above/avg/below) | Highly representative by demographics | Powered sample size | Detailed selection rationale |

**Expert Review Requirements:**
- **SME (Subject Matter Expert)**: Domain knowledge
- **Instructional Design Expert**: Learning theory, course structure
- **Media/Technology Expert**: Tools, platforms, technical implementation

**One-to-One Requirements:**
- Above-average learner (quick to master content)
- Average learner (typical learning pace)
- Below-average learner (struggles with complexity/pace)

---

### 3. Data Collection Alignment (1-5)

Do data collection instruments and methods match evaluation goals?

| Score | Alignment to Goals | Instruments Specified | Methods Appropriate | Coverage | Clarity |
|-------|-------------------|----------------------|-------------------|----------|---------|
| **1** | Vague or missing connection | No instruments described | Wrong methods (e.g., survey for usability) | Gaps; not collecting for all questions | Unclear |
| **2** | Weak connection; some mismatch | Some instruments listed | Mostly appropriate methods | Some coverage gaps | Partially clear |
| **3** | Moderate alignment | Most instruments specified | Appropriate methods | Covers most questions | Reasonably clear |
| **4** | Good alignment | Instruments attached or detailed | Well-matched methods | Comprehensive coverage | Clear specifications |
| **5** | Explicit alignment table | Complete instruments attached | Multiple complementary methods | All questions covered | Explicit connection to questions |

**Common Data Collection Methods (choose appropriately):**
- **Observation forms**: Track learner behavior, interaction patterns, confusion points (one-to-one, small group, field)
- **Think-aloud protocol**: Learners verbalize thoughts while working; reveals cognitive process (one-to-one, small group)
- **Interview guides**: Semi-structured conversations about experience, barriers, suggestions (one-to-one, small group)
- **Attitude/confidence surveys**: Likert-scale or open feedback on satisfaction, clarity, engagement (small group, field)
- **Achievement tests**: Pre/post to measure learning gains (small group, field)
- **Heuristic evaluation checklists**: Experts apply usability/accessibility heuristics (expert review)
- **Focus group discussion**: Facilitated group conversation (small group)

---

### 4. Revision Decision Framework (1-5)

Is there a clear process for turning findings into revisions?

| Score | Decision Process | Severity Ratings | Threshold Defined | Documentation | Tracking |
|-------|-----------------|------------------|------------------|----------------|----------|
| **1** | None; unclear | Not applied | None | Not documented | No tracking |
| **2** | Mentioned; vague | Partial/weak | Implied | Minimally documented | Minimal tracking |
| **3** | Process outlined | Mentioned | Somewhat clear | Documented | Basic tracking |
| **4** | Clear process documented | Severity applied | Clear threshold (e.g., 3+ learners) | Well-documented | Tracked with notes |
| **5** | Explicit Target → Data → Info → Decision | Nielsen's scale (cosmetic/minor/major/catastrophic) | Specific threshold (e.g., "if 50%+ of learners struggle, revise") | Full documentation template | Revision decision log |

**Nielsen's Severity Scale (adapted for learning):**
- **Cosmetic**: Minor annoyance; doesn't prevent learning (fix if time permits)
- **Minor**: Slight difficulty; causes minor confusion (fix before next iteration)
- **Major**: Significant barrier; some learners unable to proceed (high priority; fix before full release)
- **Catastrophic**: Prevents learning or causes wrong understanding (critical; must fix before any release)

**Decision Framework Template:**
```
Target: [What aspect evaluated? e.g., "Navigation clarity"]
Data Source: [How collected? e.g., "Think-aloud protocol from 3 learners"]
Information Gained: [What did you learn? e.g., "2/3 learners confused by menu labels"]
Revision Decision: [What to do? e.g., "Rewrite labels; test with one learner before small group"]
```

### 5. Feasibility & Practicality (1-5)

Can the evaluation be executed realistically with available resources?

| Score | Budget | Timeline | Recruitment | Facilities/Tech | Resource Plan |
|-------|--------|----------|------------|----------------|---------------|
| **1** | No budget assigned | Unrealistic (2 weeks for field trial) | No plan | None identified | None |
| **2** | Minimal budget; risky | Tight timeline; gaps | Vague recruitment | Unclear; some risk | Sketchy |
| **3** | Adequate budget identified | Realistic timeline | Recruitment plan exists | Facilities identified | Outline provided |
| **4** | Budget detailed; contingency noted | Timeline includes buffer | Clear recruitment strategy | Tech/space confirmed | Detailed resource plan |
| **5** | Fully costed; signed off | Detailed timeline with milestones | Recruitment pipeline; incentives | Fully provisioned; tested | Complete resource plan; signed off |

**Resource Estimation Guidelines:**
- **Expert Review**: 1-3 days expert time; minimal cost (~$1-3K); can be internal
- **One-to-One**: 3-5 learners × 2-3 hours each = 6-15 hours; facilitator + observer; incentive cost
- **Small Group**: 8-12 learners × 4-6 hours; facilitation, venue, incentives (~$2-5K)
- **Field Trial**: Larger sample, longer duration, real environment; highest cost and complexity

**Timeline Example (6-week evaluation):**
- Week 1: Recruit learners; prep instruments; expert review scheduled
- Weeks 2: Expert review (3 days); synthesis (2 days)
- Weeks 3: One-to-one sessions (3-5 sessions, 3 days)
- Week 4: Revise based on one-to-one findings
- Weeks 5-6: Small group evaluation (1-2 days); analysis; recommendations

### 6. Usability Engineering Integration (1-5)

Are appropriate usability methods integrated into the evaluation design?

| Score | Methods Present | Appropriateness | Coverage | Heuristics Applied | Integration |
|-------|-----------------|-----------------|----------|-------------------|-------------|
| **1** | None; learning-focus only | N/A | None | No | No integration |
| **2** | One method mentioned | Questionable fit | Limited | Minimal | Weak integration |
| **3** | One or two methods | Appropriate choice | Some coverage | Some heuristics applied | Moderate integration |
| **4** | Multiple methods | Well-chosen | Good coverage | Heuristics clearly applied | Well-integrated |
| **5** | Multiple complementary methods | Perfectly matched | Comprehensive | Nielsen's heuristics applied systematically | Seamlessly integrated |

**Usability Methods by Phase:**

| Phase | Methods | Purpose |
|-------|---------|---------|
| **Expert Review** | Heuristic evaluation against Nielsen's 10 usability heuristics; accessibility checklist; cognitive walkthrough | Identify obvious issues before involving learners |
| **One-to-One** | Think-aloud protocol; user observation; post-task interview | Understand cognitive process, pain points, confusion |
| **Small Group** | Scenario-based testing; facilitated discussion; card sorting (if applicable); System Usability Scale (SUS) | Validate findings; understand group dynamics; prioritize issues |
| **Field Trial** | In-situ observation; usage analytics (if digital); learner surveys; post-implementation interview | Assess real-world performance; identify systemic issues |

**Nielsen's 10 Usability Heuristics (adapted for learning):**
1. Visibility of system status (feedback on learning progress)
2. Match between system and real world (learner language, domain terminology)
3. User control and freedom (undo, back button, navigation clarity)
4. Consistency and standards (predictable interaction, consistent navigation)
5. Error prevention and recovery (clear instructions, helpful error messages)
6. Recognition rather than recall (visible options vs. memory demands)
7. Flexibility and efficiency (shortcuts, adjustable pacing)
8. Aesthetic and minimalist design (focused content, reduced cognitive load)
9. Help and documentation (searchable help, context-sensitive support)
10. Error diagnosis and recovery (clear identification of problems and solutions)

## System Prompt (Judge Rubric)

```
You are an expert in instructional design evaluation methodology and formative evaluation design.
Your role is to assess evaluation plans, protocols, and designs for quality and appropriateness.

You will score across 6 dimensions, each on a 1-5 scale:

1. **Phase Coverage** (1-5)
   Bordonaro's four-phase model: expert review → one-to-one evaluation → small group → field trial
   - Not all phases required for every project, but omissions should be justified
   - Expert review: Subject matter experts, instructional designers, media specialists evaluate
   - One-to-one: Individual learners from target population work through content with observation/interview
   - Small group: 8-12 learners work through, facilitated discussion, identify issues
   - Field trial: Implementation in actual learning environment with larger sample
   Score: 1=Only one phase, no justification; 5=Appropriate phases with clear rationale

2. **Evaluator Selection** (1-5)
   - Expert review requires THREE expert types: SME (content), Instruction/Learning expert, Media/Technology expert
   - One-to-one needs mixed ability learners: above-average, average, below-average from target population
   - Small group: Representative sample of target population
   - Field trial: Adequate sample size for statistical power (if applicable)
   Score: 1=Wrong evaluator types or sample; 5=Correct mix, justified selection criteria

3. **Data Collection Alignment** (1-5)
   - Instruments match evaluation goals and research questions
   - Appropriate methods: observation forms, interviews, attitude surveys, achievement tests, think-aloud, heuristic checklists
   - Data collection tools clearly specified and attached
   - Right data being collected for right questions (not collecting "nice to know")
   Score: 1=Vague data collection, misaligned to goals; 5=Specific, well-designed instruments aligned to questions

4. **Revision Decision Framework** (1-5)
   - Clear process for turning findings into revisions: Target → Data Source → Information Gained → Revision Decision
   - Severity ratings applied to prioritize fixes (Nielsen's scale: cosmetic, minor, major, catastrophic)
   - Threshold defined: what magnitude of finding triggers revision?
   - Revision decisions documented and tracked
   Score: 1=No clear decision framework; 5=Explicit severity ratings, prioritization, tracking

5. **Feasibility & Practicality** (1-5)
   - Can the evaluation be executed with available resources (budget, personnel, time)?
   - Timelines realistic given recruitment and data collection needs
   - Participant recruitment plan documented and feasible
   - Technology/facilities available for planned evaluation
   - Cost-benefit analysis: Is the evaluation commensurate with project scope?
   Score: 1=Unrealistic timeline, no resource plan; 5=Fully resourced, detailed timeline, achievable

6. **Usability Engineering Integration** (1-5)
   - Appropriate usability techniques employed: card sorting, think-aloud, heuristic evaluation, scenario testing
   - Nielsen's heuristic evaluation principles applied where appropriate
   - User testing methodology matches learning/usability goals
   - Observation protocols designed to capture usability issues
   Score: 1=No usability methods; 5=Multiple usability techniques well-integrated

Return only a valid JSON object with keys: phase_coverage, evaluator_selection, data_collection_alignment, revision_decision_framework, feasibility_practicality, usability_integration.
Each key should map to an integer 1-5.
Include a "summary" key with 1-2 sentences on overall evaluation plan quality and key missing elements.
```

### Expected Scoring Patterns
- **Rigorous evaluation plan (5-4)**: All phases included with clear rationale; appropriate evaluators; instruments attached; revision framework explicit; feasible timeline; usability methods integrated.
- **Adequate plan (3)**: Some phases; reasonable evaluators; vague instruments; weak revision framework; realistic but tight timeline; some usability methods.

- **Non-starter (1)**: Jumps to field trial only; no expert review; no instruments; no decision process; unrealistic timeline; no methodological rigor.

## Common Evaluation Scenarios

### Scenario A: Expert Review for New E-Learning Module
- **Phases**: Expert review only (SME, ID, media expert; 2-3 days)
- **Evaluators**: Internal experts + contract specialist if needed
- **Methods**: Heuristic evaluation form; cognitive walkthrough; accessibility checklist
- **Data**: Issues log; severity ratings; recommendations
- **Revision framework**: Nielsen's severity scale; target = 0 critical, <5 major issues before one-to-one
- **Timeline**: 1-2 weeks
- **Cost**: Low (internal resources)

### Scenario B: One-to-One + Small Group for Course Redesign
- **Phases**: Expert review → one-to-one (3-5 learners) → small group (8-12 learners) → revise
- **Evaluators**: 2-3 experts; learners stratified by ability; facilitator + observer for group
- **Methods**: Heuristic eval; think-aloud (one-to-one); observation forms; group discussion; post-test SUS
- **Data**: Think-aloud transcripts; observation notes; issue logs; learner feedback
- **Revision framework**: Data → decision table; threshold = 3+ learners stuck → high priority revision
- **Timeline**: 6-8 weeks
- **Cost**: Medium ($3-7K for learner incentives, facilitator time)

### Scenario C: Usability Test for LMS Implementation
- **Phases**: Expert review (technical, design, usability experts) → one-to-one task testing → small group
- **Evaluators**: 3 experts; 3-5 target end-users; 10-12 learners for validation
- **Methods**: Heuristic eval against usability standards; scenario-based task completion; think-aloud; observation; SUS survey
- **Data**: Task completion rates; error logs; think-aloud insights; SUS scores; usability issues ranked by severity
- **Revision framework**: Task completion <80% → critical fix; SUS <70 → major usability problem
- **Timeline**: 8-10 weeks
- **Cost**: Medium-high ($5-10K)

### Scenario D: Pilot Study Before Full Rollout
- **Phases**: All four (expert → 1-to-1 → small group → field trial with 50-100 learners in actual learning environment)
- **Evaluators**: Full expert panel; stratified learner sample; field facilitators
- **Methods**: All above + in-situ observation; learning analytics if digital; post-completion survey; delayed interview
- **Data**: Issue logs; achievement data; survey results; usage patterns; qualitative feedback
- **Revision framework**: Data sources → prioritized issue log; threshold triggers → must-fix vs. nice-to-have
- **Timeline**: 12-16 weeks
- **Cost**: High ($15-30K); most rigorous approach

## Validation Checklist

Use this before scoring:

- [ ] Evaluation goals/research questions explicitly stated
- [ ] Appropriate phases selected with clear justification for omissions
- [ ] Evaluator types specified for each phase (experts, learner levels, sample size)
- [ ] Data collection instruments described or attached (observation form, interview guide, survey)
- [ ] Methods match evaluation questions (not collecting extraneous data)
- [ ] Severity rating or decision criteria defined (e.g., "3+ learners affected = revise")
- [ ] Timeline realistic for phases planned and learner recruitment
- [ ] Budget allocated for incentives, facilitator, analysis time
- [ ] Usability methods integrated (heuristic eval, think-aloud, observation)
- [ ] Revision process documented (target → data source → decision)
- [ ] Stakeholder approval documented (sponsor, subject matter expert, project manager)

## Score Interpretation for Stakeholders

| Overall Range | Status | Action |
|--------------|--------|--------|
| **4.5-5.0** | Excellent | Methodologically rigorous; high confidence in findings; implement as-is |
| **3.5-4.4** | Good | Solid evaluation design; minor gaps; findings reliable; proceed with caution if major revisions needed |
| **2.5-3.4** | Fair | Adequate but weak; some methodological gaps; supplement with additional testing |
| **1.5-2.4** | Poor | Significant methodological flaws; findings questionable; redesign evaluation before proceeding |
| **<1.5** | Inadequate | Cannot proceed; evaluation must be redesigned; risk of poor implementation decisions |

## Rapid Evaluation Checklist (for quick assessments)

When time is tight, prioritize:
1. **Phase Coverage**: At least expert review + one-to-one (or expert + small group)
2. **Evaluators**: Right people for phase (don't skip expert review; recruit real learners)
3. **Instruments**: At least one data collection method per phase (observation form, interview guide)
4. **Revision Framework**: Clear severity ratings or decision rule
5. **Usability Methods**: At least think-aloud or heuristic evaluation

If these five are met, score ≥3 ("Fair"). Lower scores indicate critical gaps.
